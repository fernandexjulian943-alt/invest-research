## 判断规则

1. 只请求回答这个**具体问题**真正需要的数据，不要贪多
2. 如果已有数据足够回答，返回空列表
3. 优先请求与问题直接相关的数据类型
4. 实时行情(realtime_quote)在大多数投资问题中都有用，但如果已有则不需要重复获取
5. 如果用户问的是**全市场筛选**（如"PE最低的股票"、"哪些股票低估"、"银行行业有哪些"），需要 market_screening

## 输出格式

只返回 JSON，不要其他内容：
```json
{{"fetch": ["数据类型key1", "数据类型key2"], "reason": "简要说明为什么需要这些数据"}}
```

如果已有数据足够：
```json
{{"fetch": [], "reason": "已有数据足够回答该问题"}}
```
